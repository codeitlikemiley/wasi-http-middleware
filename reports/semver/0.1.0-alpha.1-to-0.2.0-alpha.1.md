# 0.1.0-alpha.1 to 0.2.0-alpha.1 semver report

- Baseline revision: `6977a75c4d2cb1558158341c109e419189dc2c52`
- Tool: `cargo-semver-checks 0.48.0`
- Candidate: `0.2.0-alpha.1`
- Decision: intentional breaking alpha update; every detected removal below is
  documented in `MIGRATION.md`.

Forced-minor comparison is used so the tool enumerates breaks that the normal
0.1-to-0.2 major-change classification intentionally permits.

## `wasi-http-metadata`

Four breaking lint classes were reported:

- removed `Principal`;
- removed `parse_principal` and `insert_principal`;
- removed `AUTH_SUBJECT_HEADER`, `AUTH_ISSUER_HEADER`, and
  `AUTH_SCOPES_HEADER`;
- removed `MetadataError::InvalidIdentityValue`,
  `MetadataError::InvalidScopes`, and `MetadataError::InvalidHeader`.

## `wasi-http-policy-core`

Three breaking lint classes were reported:

- removed `PolicyRequest` and `PolicySuccess`;
- removed `parse_policy_response`;
- removed `AuthDecision::Forbidden`.

## `wasi-http-middleware-component-support`

The forced-minor comparison passed all 196 applicable checks. This crate was
private in 0.1 and becomes publishable in 0.2, so the check is retained as a
future public-API baseline.
